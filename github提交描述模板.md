# GitHub 提交描述模板

后续每次提交说明，默认按这个模板来写。  
目标：

- 准确
- 简洁
- 不空泛
- 不夸大

## 1. 完整模板

```text
type: 简短说明本次改动

本次已解决：
- 
- 

本次已确认：
- 

当前仍未完全解决：
- 

说明：
本次提交属于阶段性改动，不代表相关功能已经最终闭环。
```

## 2. 小提交模板

如果这次只是一个很小的修改，可以用简化版：

```text
type: 简短说明本次改动

本次已解决：
- 

说明：
本次提交属于阶段性改动。
```

## 3. 常用 type

```text
feat: 新功能
fix: 修复问题
refactor: 重构整理
docs: 文档更新
test: 测试相关
chore: 配置/脚本/环境调整
style: 纯格式调整
```

## 4. 写法要求

- 标题要说明“这次主要改了什么”
- 不要只写“修复问题”“完善功能”这种空话
- `本次已解决` 只写已经实际处理的内容
- `本次已确认` 只写已经运行或联调验证到的现象
- `当前仍未完全解决` 用来写遗留问题，避免把阶段性结果写成最终完成
- 如果本次只是小改动，可以不写“本次已确认”和“当前仍未完全解决”

## 5. 示例

```text
fix: 修正本地联调中的 Kafka 与前端代理问题

本次已解决：
- 调整 ZooKeeper 本地数据目录，修复 Windows 环境下启动不稳定的问题。
- 将 Nginx 转发收敛为单后端实例，修复页面加载不完整的问题。

本次已确认：
- Nginx、ZooKeeper、Kafka 的本地基础启动链路已打通。

当前仍未完全解决：
- Kafka 异步秒杀消息消费后的订单落库链路仍需继续验证。

说明：
本次提交属于阶段性改动，不代表相关功能已经最终闭环。
```
